# product_writer 工作约定

1. 不要修改 `C:\Users\haixiang\Desktop\batchword`。`product_writer` 是独立新项目，和 `batchword` 没有代码关系。
2. 第一阶段只做“标题 + 提示词 -> AI 文章 -> Word 排版 -> 加粗 -> 基础质检”，不做插图、封面、PDF、网页界面、复杂资料库、数据库、多模型 UI，也不做护眼灯迁移。
3. 新产品优先新增或修改 `projects/<产品id>/`，不要为了具体产品资料修改核心 Python 代码。
4. 不要把具体产品资料、品牌资料、推广内容写死进 Python。
5. 品牌白名单是最高命令：正文中的所有品牌名和产品名必须来自 `projects/<产品id>/brands.txt`；该文件不能为空，主推产品也必须包含在白名单中。
6. Word 输出必须全文宋体 12pt，中文、英文、数字都必须统一宋体。
7. Word run 必须设置 `run.font.name = "宋体"`，并同时设置 `w:ascii`、`w:hAnsi`、`w:eastAsia` 为 `宋体`。
8. 所有段落段前、段后必须是 0 pt，避免用户手动删除段落间距。
9. 全文必须是普通段落，禁止使用 Word 标题样式或标题段落 API。
10. 禁止居中、标题大字号、首行缩进、1.5 倍行距；必须使用单倍行距。
11. 标题、结构标题、推荐行、terms 关键词只允许加粗，不允许改变字号、字体、对齐、段距。
12. 不要使用额外空段落制造间距。
13. 中文输出注意设置 `PYTHONIOENCODING=utf-8`。
